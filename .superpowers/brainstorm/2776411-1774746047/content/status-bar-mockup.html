Status Bar Design: What goes in the header?

This bar sits at the top of the existing dashboard, next to the LIVE indicator. Click the one that feels right.

SCORCHED LIVE
yfinance FRED Polygon Finnhub Claude Alpaca EDGAR DETAILS

A: Individual Dots

Each API gets its own dot + label. Green/yellow/red. You see exactly which service is having issues at a glance. Yellow = degraded (>20% failures), Red = down (>50% or last call failed).

SCORCHED LIVE
APIs 5 / 1 / 1 Broker ALPACA-PAPER SYSTEM

B: Grouped Summary

Aggregated view: "APIs 5/1/1" (5 green, 1 yellow, 1 red) + broker status badge. Cleaner, but you need to click through to see which specific API is the problem.

SCORCHED LIVE
6/7 SYS

C: Dots Only

Just colored dots in a row — no labels. Hover to see which is which. Maximum compactness. "6/7" tells you the count. All green = don't even notice it. Any yellow/red = something to look at.

My recommendation: A (Individual Dots) — when something goes yellow/red, you want to know instantly which API without clicking. The labels are tiny and don't take much space.
